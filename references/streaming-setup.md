# Setup Server-Sent Events (SSE) Streaming in FastAPI

Currently, the `/query` endpoint in `src/api/v1/routes/query.py` returns the final response synchronously. To improve the user experience, especially for long-running LLM tasks or LangGraph agents, you can implement SSE (Server-Sent Events) using FastAPI's `StreamingResponse`.

Follow these steps to implement streaming in your Reranking RAG System:

## 1. Modify the Agent (`src/api/v1/agents/agents.py`)

LangGraph and LangChain support streaming natively via `.astream()` or `.astream_events()`. You need to create an asynchronous generator that yields tokens or events.

```python
import json

async def run_search_agent_stream(query: str):
    # Assuming `graph` is your compiled LangGraph
    # You will use `astream` or `astream_events` to get chunks
    async for event in graph.astream_events({"messages": [("user", query)]}, version="v1"):
        kind = event["event"]

        # If it's a token generated by the chat model
        if kind == "on_chat_model_stream":
            content = event["data"]["chunk"].content
            if content:
                # Format as an SSE data stream payload
                yield f"data: {json.dumps({'token': content})}\n\n"

    yield "data: [DONE]\n\n"
```

## 2. Update the Service Layer (`src/api/v1/services/query_service.py`)

Create a new service function that calls the streaming agent instead of the synchronous one.

```python
from src.api.v1.agents.agents import run_search_agent_stream

async def query_documents_stream(query: str):
    # Just return the async generator
    return run_search_agent_stream(query)
```

## 3. Update the Router (`src/api/v1/routes/query.py`)

Import `StreamingResponse` from `fastapi.responses` and create an endpoint that returns the stream stream.

```python
from fastapi.responses import StreamingResponse
from src.api.v1.services.query_service import query_documents_stream

@router.post("/query/stream")
async def stream_query_endpoint(request: QueryRequest):
    """
    Endpoint that returns an SSE stream of the agent's response.
    """
    generator = await query_documents_stream(request.query)

    return StreamingResponse(
        generator,
        media_type="text/event-stream"
    )
```

## 4. How to Consume the Stream (Client Side)

The client needs to read the chunks as they come in. Here is a simple example using the native `fetch` API:

```javascript
async function fetchStream() {
  const response = await fetch("/api/v1/query/stream", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ query: "tell me about agentic rag" }),
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder("utf-8");

  while (true) {
    const { value, done } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    console.log("Received chunk:", chunk);
    // Parse the SSE payload (e.g., extracting data: {...})
  }
}
```

## Summary

1. Use `StreamingResponse` from FastAPI with `media_type="text/event-stream"`.
2. Convert your LangGraph agent execution to use async methods like `astream_events()`.
3. Format the yielded string correctly according to the SSE standard (`data: <json>\n\n`).
